# BC-IDPR

**Boundary Compensation — Independent Deformation and Phase-Resolved Response Programme**

Status: `PROGRAMME_DRAFT`
Version: `v0.1.0-draft`
Dependency order: `P3 -> certification -> P1 -> synthesis/null result`

## Executive thesis

A phase-resolved residual cannot be attributed specifically to a deformation parameter until the programme constructs and certifies a path that changes that parameter while keeping normalized geometry, representation scale, branch identity, and the observation protocol fixed within declared tolerances.

## Programme map

- `manifesto/` — branch charter, claim boundary, publication sequence.
- `governance/` — naming rules, document registry, stage gates, audit records.
- `p3-independent-deformation/` — construction and certification of independent deformation families.
- `p1-phase-resolved-response/` — preregistered signed/complex residual experiments.
- `shared/` — notation, benchmark registry, failure states, theorem-obligation graph.
- `audits/` — Project Steward and Publication Auditor reports.

## Non-negotiable order

1. Define `q`, representation scale `rho`, normalized geometry `xi`, and protocol variables `vartheta`.
2. Construct an exact or tolerance-certified P3 deformation.
3. Pass the parameter-separation gate.
4. Freeze the P1 preregistration.
5. Run the phase scan without changing predictors or exclusions.
6. Publish either a robust result or a valid null result.

## Claim firewall

The programme may establish mathematical deformation families, parameter-separation bounds, finite-resolution observables, reproducible phase structure, obstructions, and null results. It does not infer fundamental dynamics, physical time, particle masses, spacetime geometry, or a unique hidden sector from modulation alone.
