# RC02 Confirmatory Change Control

After the unseal certificate is issued, the following are immutable: the eight families, the 106-carrier derived scope, `n_star=5`, control frequency `9.5`, grid, baseline, observable definitions, aggregation, thresholds, sign test, phase statistics, quality-control thresholds and A/B/C/D precedence.

Any scientific change requires a new contract identifier. Code-only correction before reading outputs requires a documented hash change and a fresh unseal certificate. Once any confirmatory response has been computed, no code, threshold or endpoint change is admissible inside RC02.

Confirmatory data may not be used to create an alternative mode, subgroup, family exclusion or post hoc predictor. Exploratory analyses must be labeled non-confirmatory and moved to a new contract after the frozen result is reported.

No statement from the Gemini advisory report is used as evidence.
