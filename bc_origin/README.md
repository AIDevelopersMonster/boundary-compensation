## Purpose

BC-Origin is the origin-oriented branch of the Boundary Compensation program. Its purpose is to test whether observable elementary structures can be modeled as non-fundamental shadows of hidden residual generators.

The program does not start by claiming that a specific hidden sector is physically real. It starts with a stricter and more useful task: construct minimal models in which an observable structure is generated by a hidden structural equation, and where its admissible scale or initial data are selected by compatibility conditions rather than assigned as free cutoffs.

## Program distinction

BC-Certification asks whether a finite-resolution observed claim is stable under thresholds, atlases, quotient choices, and perturbations.

BC-Origin asks whether an observed elementary layer can arise as a projected shadow of hidden compensation residues.

In short:

```text
BC-Certification: observed structure -> stability certificate
BC-Origin: hidden residual generator -> observable shadow
```

## First practical target

The first practical target is a minimal resonant-shadow model with oriented winding.

A hidden generator is represented by an index-winding pair

```text
h = (q, n)
```

where `q` is an index-like residual charge and `n` is an oriented winding number. The observable shadow separates the hidden winding into two channels:

```text
scale channel: |n|
sign channel:  sign(n)
```

The one-generator closure equation is

```text
(lambda_q(ell) * L) + theta(q) = 2*pi*|n|
lambda_q(ell) = mu_q / ell
```

This gives the observable scale ratio

```text
ell / L = mu_q / (2*pi*|n| - theta(q))
```

The two-generator signed-shadow model uses

```text
s = sign(n1*n2)
D_signed = [[d1 + gamma*s, kappa],
            [kappa,        d2 + gamma*s]]
```

with three core effects:

1. orientation-controlled shadow localization;
2. admissibility horizon;
3. coupling-induced shadow gap.

## Research posture

The aim is not a maximally cautious minimal-rank answer. The aim is the strongest mathematically admissible model, with explicit status labels for each claim:

- **proved inside the toy model**;
- **structural interpretation**;
- **physics-motivated but not derived**;
- **speculative extension**;
- **failure mode / critic target**.

The program should seek maximal strength under explicit constraints, not rhetorical safety.
