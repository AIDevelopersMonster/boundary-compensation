# BC-Origin Program Definition

## Canonical name

**BC-Origin** / **BC-O**  
**Boundary Compensation Origin Program**

## Core question

Can an observed elementary structure be modeled as a non-fundamental shadow of a hidden residual generator, with its admissible scale, orientation sign, and local observability selected by structural compatibility equations?

The aim is not to identify a final physical hidden sector. The aim is to construct a controlled class of models in which the observed layer is generated as a shadow layer rather than assumed as fundamental.

## Minimal positive thesis

A hidden generator is represented by an index-winding pair

```text
h = (q, n)
```

where:

- `q` is an index-like residual type;
- `n in Z` is an oriented winding number.

The oriented winding decomposes into two observable channels:

```text
scale channel: |n|
sign channel:  sign(n)
```

An observable elementary shadow is a projected object

```text
s = Pi(h)
```

whose scale ratio is selected by a closure equation.

## One-generator model

The first model class uses a spectral closure equation:

```text
lambda_q(ell) * L + theta(|q|) = 2*pi*|n|
lambda_q(ell) = mu_|q| / ell
```

The selected observable scale is

```text
ell / L = mu_|q| / (2*pi*|n| - theta(|q|))
```

The sign channel is

```text
sigma = sign(n)
```

## Two-generator signed-shadow model

For two generators

```text
h1 = (q1, n1)
h2 = (q2, n2)
```

define the orientation product

```text
s12 = sign(n1*n2)
```

and the signed common-shift operator

```text
D_signed = [[d1 + gamma*s12, kappa],
            [kappa,          d2 + gamma*s12]]
```

where

```text
di = 2*pi*|ni| - theta(|qi|)
```

This produces three core toy-level effects:

1. orientation-controlled shadow localization;
2. admissibility horizon;
3. coupling-induced shadow gap.

## Structural coupling requirement

BC-Origin should not leave `gamma` and `kappa` as unexplained fitted numbers. Later stages should replace them with structural values generated by equations, kernels, overlap integrals, or selection rules.

A candidate for BC-Origin II is

```text
kappa_ij = <u_ni, A u_nj>
```

where

```text
u_n(phi) = exp(i*n*phi)/sqrt(2*pi)
A = (1 - Delta_S1)^(-1)
```

leading to

```text
kappa_ij = 1/(1 + (ni-nj)^2)
```

## Strong-result principle

The project should seek the strongest valid model under explicit constraints. Each claim must be labeled as one of:

1. theorem inside the toy model;
2. computational result;
3. structural interpretation;
4. physics-motivated analogy;
5. speculative extension;
6. critic target.

## Immediate success criterion

BC-Origin I succeeds if it proves and computes:

1. oriented winding decomposes into scale and sign channels;
2. closure selects observable scale without a manual cutoff;
3. two shadows produce signed spectral localization effects;
4. horizon crossing marks loss of localized observable shadow;
5. coupling produces a reproducible branch gap;
6. all free couplings are either structurally defined or explicitly marked as placeholders for BC-Origin II.
