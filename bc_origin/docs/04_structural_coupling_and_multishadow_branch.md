# BC-Origin II: Structural Coupling and Multi-Shadow Geometry

## Status

This document defines the next research branch after BC-Origin I.

BC-Origin I fixes the one- and two-shadow signed spectral model. BC-Origin II addresses the next obstacle: couplings such as `kappa` and `gamma` must not be inserted as free numerical parameters. They should be generated by structural equations, overlap operators, kernels, or selection rules.

## Main problem

The signed two-shadow model uses

```text
D_signed = [[d1 + gamma*s, kappa],
            [kappa,        d2 + gamma*s]]

s = sign(n1*n2)
```

This produces three toy-level effects:

1. orientation-controlled shadow localization;
2. admissibility horizon;
3. coupling-induced shadow gap.

The weak point is that `gamma` and `kappa` are still externally chosen unless they are derived from a structural operator.

## Constitutional rule

No coupling coefficient should enter BC-Origin as an unexplained fitted number.

Acceptable sources of coupling:

1. overlap integrals of hidden winding modes;
2. Fourier coefficients of a fixed boundary kernel;
3. Green/resolvent kernels of canonical circle operators;
4. selection rules induced by phase closure;
5. normalized adjacency operators generated by the hidden residual structure.

Unacceptable source:

```text
choose kappa = numerical value by hand
```

## Minimal structural coupling candidate

Let hidden winding modes be

```text
u_n(phi) = exp(i*n*phi)/sqrt(2*pi)
```

on the circle. Let a canonical boundary operator be

```text
A = (1 - Delta_S1)^(-1)
```

with Fourier coefficients

```text
a_m = 1/(1 + m^2)
```

Then define the coupling between two windings as

```text
kappa_ij = <u_ni, A u_nj> = a_(ni-nj)
         = 1/(1 + (ni-nj)^2)
```

This is no longer a fitted constant. It is a structural overlap value generated by the circle operator.

An oriented signed version is

```text
g_ij = sign(ni*nj) * a_(ni-nj)
```

This separates:

```text
amplitude channel: a_(ni-nj)
sign channel:      sign(ni*nj)
```

## Multi-shadow operator

For `N` shadows, define

```text
d_i = 2*pi*|n_i| - theta(|q_i|)
a_ij = 1/(1 + (n_i-n_j)^2)
s_ij = sign(n_i*n_j)
g_ij = s_ij * a_ij
```

A minimal structural multi-shadow operator can be written as

```text
D_ii = d_i + sum_{j != i} g_ij
D_ij = a_ij, i != j
```

or, for an oriented deformation version,

```text
D_ii = d_i + sum_{j != i} s_ij*a_ij
D_ij = s_ij*a_ij, i != j
```

The second version must be checked for gauge equivalence and sign loss.

## Important correction: no automatic spin-glass frustration

The pair sign

```text
s_ij = sign(n_i*n_j) = sign(n_i)*sign(n_j)
```

is a rank-one sign structure. Therefore every cycle satisfies

```text
s_ij*s_jk*s_ki = +1
```

for nonzero windings. This means the sign graph is balanced and does not by itself create genuine spin-glass frustration.

A three-shadow example with signs `(+,-,-)` gives pair signs `(-,+,-)`, whose triangle product is positive. This can produce nontrivial localization patterns, but it is not mathematical frustration in the strict spin-glass sense.

Genuine frustration requires an additional non-factorizable edge structure, such as:

```text
epsilon_ij not equal sigma_i*sigma_j
```

or a cycle holonomy

```text
epsilon_ij*epsilon_jk*epsilon_ki = -1
```

This is a candidate for BC-Origin III, not a theorem of the current sign-product model.

## Good news that survives criticism

The strong result is not spin-glass frustration. The strong result is:

```text
hidden oriented winding -> sign channel + scale channel
```

and, with structural coupling,

```text
circle operator overlap -> non-arbitrary coupling matrix
```

Together they produce a multi-shadow spectral geometry in which observable scale localization is generated by structural winding data rather than fitted constants.

## BC-Origin II target theorem

A possible theorem for BC-Origin II:

Given hidden winding modes on `S1` and the canonical resolvent kernel `A=(1-Delta_S1)^(-1)`, the induced overlap matrix

```text
K_ij = 1/(1 + (n_i-n_j)^2)
```

is a fully structural coupling matrix. When combined with orientation signs

```text
s_ij = sign(n_i*n_j)
```

it defines a multi-shadow operator whose eigen-denominators and admissibility regions are determined entirely by winding data and closure denominators.
