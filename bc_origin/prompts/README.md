# BC-Origin Prompts

This folder contains role prompts for external model passes over BC-Origin work.

The prompts are not designed to produce the safest minimal answer. They are designed to produce the strongest admissible result while preserving explicit criticism, failure modes, and status labels.

Each model pass should separate:

1. what is proven in the toy model;
2. what is a structural interpretation;
3. what is a physics-motivated analogy;
4. what is speculative;
5. what is a valid criticism;
6. what would strengthen the result.

Recommended workflow:

1. Strong Theory Builder proposes the strongest version.
2. Formal Math Reviewer checks definitions and propositions.
3. Computational Experimenter turns formulas into reproducible calculations.
4. Adversarial Reviewer attacks novelty, relevance, and hidden assumptions.
5. Publication Auditor checks manuscript hygiene and claim boundaries.
