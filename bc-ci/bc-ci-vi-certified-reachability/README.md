# BC-CI VI Certified Reachability Demo

This supplement accompanies the v0.1.1 clean release:

**Boundary Compensation - Compensated Islands VI: Certified Reachability and Pre-Causal Order**

It audits a finite directed diagnostic graph. Nodes are declared diagnostic points. Edges are declared local continuation attempts. The script marks robustly certified edges, computes certified reachability, detects strongly connected components, builds the quotient order, and labels cycles.

This is a certification demo only. It is not physical spacetime, not a causal simulation, not a light-cone model, not dynamics, not a Hamiltonian evolution, and not an empirical prediction.

## Interpretation

The demo supports the BC-CI VI status discipline:

- `ROBUST_EDGE`: edge supported by at least one accepted robust hidden section and not reset.
- `CERTIFICATE_RESET`: edge cannot be silently continued.
- `TUNED_EDGE`: edge is supported only by fragile or tuning-artifact sections.
- `CERTIFIED_CYCLE`: cycle entirely inside the robust edge graph.
- `TUNED_LOOP`: cycle uses at least one tuned or fragile edge.
- `RESET_CYCLE`: cycle uses at least one reset edge.
- `QUOTIENT_COMPONENT`: strongly connected component of the robust reachability preorder.

Expected JSON outputs:

- `reachability_matrix`: finite certified reachability relation generated by robust edges.
- `strongly_connected_components`: mutual-reachability components of the robust graph.
- `quotient_order`: directed order relation between quotient components.
- `edge_status_table`: per-edge certification status.
- `cycle_status_table`: audited directed cycles and their statuses.
- `reset_barrier_report`: edges blocked by `CERTIFICATE_RESET`.
- `tuned_loop_audit`: cycles depending on tuned or fragile support.

The central output is a finite preorder and its quotient partial order. It is not physical causality. The demo uses only the Python standard library; NetworkX may be used by external implementations but is not required here.
